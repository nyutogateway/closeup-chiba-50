# クローズアップCHIBA50 ｜ WordPress版 納品ガイド

チバテレ「クローズアップCHIBA50」リニューアルサイトの WordPress テーマ一式に関する運用マニュアルです。
掲載企業の追加・編集、公開手順、注意点をまとめています。

---

## 1. 概要

- 静的デザインを、**前任テーマと同じ構成の作法**で WordPress 化したものです。
- 掲載企業は **WordPressの「投稿」** として管理します（1社＝1投稿）。
- 記事本文以外の情報（メイン画像・ロゴ・代表者・星 など）は **カスタムフィールド**で入力します。
- デザイン（見た目）は静的版から**一切変更していません**。

### ページ構成
| ページ | テンプレート | 内容 |
|---|---|---|
| トップ | `front-page.php` | ヒーロー演出＋CONCEPT＋LEADERS（50社をスライダーで表示） |
| 記事詳細 | `single.php` | 各企業の記事（★評価・代表者・本文・前後記事リンク） |
| お問い合わせ | `page-contact.php` | 固定ページ |
| プライバシーポリシー | `page-policy.php` | 固定ページ |
| 404 | `404.php` | トップへ自動リダイレクト |

---

## 2. 使用プラグイン

- **Smart Custom Fields**（必須）… カスタムフィールドの入力に使用。**有効化してください**。
  - ※「Advanced Custom Fields / Secure Custom Fields」とは**別物**です。混同注意。
  - **カスタムフィールド系プラグインは1つだけ有効**にしてください（複数同時有効化は不可）。

---

## 3. テーマの導入・有効化

1. 管理画面 **外観 → テーマ → 新規追加 → テーマのアップロード**
2. `wp-theme.zip` を選択 → インストール
3. **有効化**

> テーマフォルダ名は任意です（`BASE_URL` は自動でテーマの場所を指すため、名前を変えても動作します）。

---

## 4. 掲載企業の追加・編集

**投稿 → 新規追加**（または既存投稿を編集）で、以下を入力します。

### 記事タイトル・本文
- **タイトル**＝会社名（例：`イオンアグリ創造株式会社`）
- **本文**＝記事の内容（見出し・段落・画像を通常どおり入力）

### カスタムフィールド（投稿画面の「設定」欄）
| ラベル | 用途 | 例 |
|---|---|---|
| メインイメージ | 記事上部の大きな画像 | 代表者などの写真 |
| 会社ロゴ | トップ一覧カードのロゴ | 企業ロゴ |
| 記事見出し（キャッチコピー） | カード／記事上部の見出し | 「サプライチェーン一貫型農業ビジネス…」 |
| 代表氏名 | 代表者の氏名 | 福永 庸明 |
| 代表役職 | 代表者の役職 | 代表取締役社長 |
| 代表のプロフィール | COMPANY DATA欄の経歴 | 「1995年入社…」 |
| 企業URL | 公式サイトへのリンク | https://… |
| 代表者 | （氏名の別入力欄。氏名が入っていればそちら優先） | |
| **星の数** | 注目度（★の数） | `3`＝★★★／`2`＝★★／`0`＝なし |

> **星について**：`3` `2` `0` から選択します。トップのカードと記事上部に★として表示されます（`0`は非表示）。

---

## 5. 掲載順の変更

- 現在はトップの表示順・グループ分けが **投稿の並び順** で決まります（上から約7社ずつ GROUP 01〜08）。
- ドラッグ＆ドロップで並べ替えたい場合は、プラグイン **「Intuitive Custom Post Order」** 等を導入すれば、投稿一覧での並べ替えがそのままトップに反映できます（別途ご相談ください）。

---

## 6. 固定ページ（お問い合わせ・プライバシーポリシー）

1. **固定ページ → 新規追加**で「お問い合わせ」「プライバシーポリシー」を作成
2. 各ページの **テンプレート**を選択
   - お問い合わせ → 「お問い合わせページテンプレート」
   - プライバシーポリシー → 「プライバシーポリシーページテンプレート」
3. スラッグをそれぞれ `contact` / `policy` に設定（ヘッダー・フッターのリンク先に合わせるため）

---

## 7. 画像について

- 記事の画像は、通常どおり**メディアライブラリ**にアップして使えます。
- 今回の初期データ投入（インポート）分は、`/wp-content/uploads/_import_photos/` に置いた画像を参照しています。
  - **この画像フォルダは削除しないでください**（削除すると初期投入分の画像が表示されなくなります）。
  - 将来きちんとメディアライブラリで管理し直すことも可能です（ご相談ください）。

## 8. 本番（chiba-tv）公開時の設定

現在はステージング（テスト環境）で動く設定です。本番公開時に **`functions.php` の2〜3行**を調整します。

```php
// 本番公開時：ナビ・画像も chiba-tv に固定する場合
define('BASE_URL', 'https://www.chiba-tv.com/closeup/wp-content/themes/＜テーマ名＞');
define('HOME_URL', 'https://www.chiba-tv.com/closeup/');
```

- **canonical** は既に常に `https://www.chiba-tv.com/closeup/...` を出力する設定です（`PROD_URL`）。ステージングで動いていても正規URLは chiba-tv を指します。
- 本番が `/closeup/` 配下のため、初期投入画像のパス（`/wp-content/…`）を `/closeup/wp-content/…` へ合わせる必要があります（公開時に対応します）。

> 本番切り替えの実作業は制作側で対応しますので、公開タイミングをお知らせください。


## 10. 注意事項・既知の制限

- **カスタムフィールド系プラグインは1つだけ有効**に（Smart Custom Fields のみ推奨）。
- 初期投入した画像は URL 参照のため、**Smart Custom Fields の管理画面上では画像フィールドが空に見える**場合があります（フロント表示は正常）。今後の編集で画像を再設定すれば通常のメディア管理になります。
- デザイン（CSS/レイアウト）は確定済みです。**変更が必要な場合は制作側で対応**します（管理画面からの編集ではデザインは変わりません）。
